Engenharia de contexto. O que é esse negócio? Contexto é o novo código, pelo menos essa é a tese que eu tenho. Se você tem um bom contexto, você vai conseguir delegar melhor as tarefas para inteligência artificial, muito melhor do que se você tiver um bom código. Contexto é mais importante do que o código, porque o contexto é o que é inteligência artificial. Então o que é engenharia de contexto? É uma disciplina de decidir, para cada chamado ao modelo, exatamente o que que entra na janela de contexto. Quais instruções, quais docs, qual histórico, quais tools, em que ordem e com que peso. Se prompt engineering é o como perguntar, o context engineering é o que colocar ao lado da pergunta. A mudança de paradigma principal aqui é o seguinte, a gente vem da era do prompt. Ou eu escrevo um prompt perfeito, o modelo responde de memória, a única mensagem é estática e o conhecimento travado no cutoff, ou seja, o treinamento LLM é o que define o conhecimento dele. Se teve um cutoff e ele não pegou uma versão específica do framework que você está utilizando, ele não sabe. Um exemplo clássico, quando você estava ali no cutoff de conhecimento do Next.js 13 para o 12, ou seja, o 12 era uma maneira de você fazer e o 13 mudou, muitas vezes o LLM alucinava quando estava trabalhando com o Next.js 13. Beleza? Então esse era um caso clássico disso. Então o problema do prompt, somente o prompt, é que ele não consegue trazer essas questões de conhecimento, as questões de conhecimento específicas de uma base de código. Talvez, não estamos nem falando de cutoff, talvez a gente está falando de uma base de código que usa uma versão legada de um framework. E aí o LLM gera coisas que tentam embutir ali coisas que não existem naquela versão atual. Ou seja, como é que a gente resolve isso? Com engenharia de contexto. Então eu vou fazer a curadoria com o contexto certo, o modelo vai responder com os dados frescos, eu vou trazer uma montagem dinâmica por requisição e o conhecimento sempre vai estar atualizado. À medida que a minha base de código evolui, o meu contexto evolui junto com ela. Você consegue escalar bem e você consegue criar uma infra de contexto reusável. A gente tem sete camadas do contexto, qual que é o stack aqui? A primeira delas é o system prompt, que é a identidade, as regras globais, o tom, ele raramente vai mudar, ele está sempre presente. O fluxo do Context Assembly, como é que o Coding Agent vai assimilar esse contexto?
