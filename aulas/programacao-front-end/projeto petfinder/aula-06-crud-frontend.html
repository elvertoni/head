Seção 01

Introdução

O sistema da Aula 05 já cadastra e valida, mas só conhece duas das quatro operações de um sistema real. Hoje completamos o CRUD: editar pets já cadastrados, excluir, buscar pelo nome e filtrar por espécie. É nesta aula que o projeto deixa de parecer brinquedo e começa a parecer software de verdade — mesmo com os dados ainda na memória do navegador.

Seção 02

O que vamos construir

✓

Entrega da aula

Cada card de pet ganha dois botões — Editar e Excluir. Acima da lista, um campo de busca por nome e uma lista suspensa para filtrar por espécie, mais um contador de quantos pets estão visíveis. Clicar em Editar preenche o formulário com os dados do pet, troca o botão para Salvar alterações e mostra um Cancelar edição.

i

Antes de começar

O projeto da Aula 05 precisa estar funcionando: cadastro com validação e mensagens em verde e vermelho. Mantenha o DevTools aberto — esta é a aula com mais código do bimestre, e o console é o seu melhor amigo.

Seção 03

O que é CRUD

i

As quatro operações de qualquer sistema

CRUD é a sigla em inglês para Create, Read, Update, Delete — criar, ler, atualizar e excluir.

Criar é cadastrar um item novo (temos desde a Aula 04). Ler é listar os itens (temos). Atualizar é mudar dados de um item já cadastrado e Excluir é apagar um item — as duas que faltavam e que entram hoje. Quase todo sistema corporativo do mundo (cadastro de clientes, controle de estoque, lista de tarefas, agenda) é um CRUD por baixo. Dominar esse padrão é dominar o esqueleto de centenas de telas profissionais.

4. JS parte 2 — o submit decide entre cadastrar e atualizar

Substitua a função do submit por esta versão, que olha para editandoId antes de decidir:

javascript 
formulario.addEventListener('submit', function (evento) {
  evento.preventDefault();
  limparErrosVisuais();

  const pet = montarPetDoFormulario();
  const erro = validarPet(pet);
  if (erro) { mostrarMensagem(erro, 'erro'); return; }

  if (editandoId) {
    atualizarPet(editandoId, pet);
    mostrarMensagem('Pet atualizado!', 'sucesso');
  } else {
    cadastrarPet(pet);
    mostrarMensagem('Pet cadastrado!', 'sucesso');
  }

  limparFormulario();
  mostrarPetsNaTela();
});

Acrescente as duas funções que o submit chama:

javascript 
function cadastrarPet(pet) {
  pets.push({ id: Date.now(), ...pet });
}

function atualizarPet(id, dados) {
  pets = pets.map(function (p) {
    return p.id === id ? { id, ...dados } : p;
  });
}

5. JS parte 3 — filtros, card com ações e helpers

Substitua a mostrarPetsNaTela por esta versão, que aplica busca e filtro:

javascript 
function mostrarPetsNaTela() {
  listaPets.innerHTML = '';
  const termo = campoBusca.value.toLowerCase();
  const especie = filtroEspecie.value;

  const filtrados = pets.filter(function (p) {
    const nomeOk = p.nome.toLowerCase().includes(termo);
    const especieOk = !especie || p.especie === especie;
    return nomeOk && especieOk;
  });

  atualizarContador(filtrados.length);
  for (const pet of filtrados) listaPets.appendChild(criarCardPet(pet));
}

Adicione a criarCardPet, que monta o card com os botões e liga os cliques:

javascript 
function criarCardPet(pet) {
  const card = document.createElement('article');
  card.classList.add('card-pet');
  const imagem = pet.foto || 'https://placehold.co/600x400?text=Sem+foto';
  const classeStatus = pet.status === 'Adotado' ? 'status adotado' : 'status';

  card.innerHTML = `
    <img src="${imagem}" alt="${pet.nome}">
    <div class="card-pet-conteudo">
      <h3>${pet.nome}</h3>
      <p>Espécie: ${pet.especie}</p>
      <p>Cidade: ${pet.cidade || 'Sem cidade'}</p>
      <span class="${classeStatus}">${pet.status}</span>
      <div class="acoes-card">
        <button class="botao-editar" type="button">Editar</button>
        <button class="botao-excluir" type="button">Excluir</button>
      </div>
    </div>
  `;

  card.querySelector('.botao-editar').addEventListener('click', () => carregarPetParaEdicao(pet.id));
  card.querySelector('.botao-excluir').addEventListener('click', () => excluirPet(pet.id));
  return card;
}

A função que carrega um pet no formulário para edição:

function carregarPetParaEdicao(id) {

  document.querySelector('#foto').value = pet.foto;
  document.querySelector('#descricao').value = pet.descricao;
  document.querySelector('#status').value = pet.status;
  botaoSalvar.textContent = 'Salvar alterações';
  botaoCancelar.classList.remove('escondido');

E, por fim, exclusão, limpeza do formulário e contador:

javascript 
function excluirPet(id) {
  const pet = pets.find(function (p) { return p.id === id; });
  if (!pet) return;
  if (!confirm(`Deseja excluir o cadastro de ${pet.nome}?`)) return;
  pets = pets.filter(function (p) { return p.id !== id; });
  mostrarPetsNaTela();
  mostrarMensagem('Pet excluído.', 'sucesso');
}

function limparFormulario() {
  formulario.reset();
  campoId.value = '';
  editandoId = null;
  botaoSalvar.textContent = 'Cadastrar pet';
  botaoCancelar.classList.add('escondido');
  limparErrosVisuais();
}

function atualizarContador(total) {
  if (total === 0) contadorPets.textContent = 'Nenhum pet encontrado.';
  else if (total === 1) contadorPets.textContent = '1 pet encontrado.';
  else contadorPets.textContent = `${total} pets encontrados.`;
}
Seção 05

Checkpoint e desafio

★

Sua página passa se…

Três pets de espécies diferentes geram três cards com botões Editar e Excluir; digitar parte do nome filtra os cards em tempo real; escolher uma espécie reduz a lista; clicar em Editar preenche o formulário, troca o botão para Salvar alterações e mostra o Cancelar edição; e clicar em Excluir pede confirmação, remove o card e atualiza o contador.

★

Desafio (até 12 min)

Adicione um terceiro filtro, por status: junto dos outros, um select com Todos, Disponível e Adotado. Na mostrarPetsNaTela, combine os três filtros (nome, espécie e status) com o operador &&. Teste cadastrando pets em status diferentes e marcando alguns como Adotado pelo botão Editar.

Seção 06

Erros comuns

!

Editar passa a mexer sempre no mesmo pet

Sintoma: você edita o pet A, salva, depois clica em Editar do pet B — mas, ao salvar, quem muda é o A de novo. Causa: faltou zerar editandoId na limparFormulario; o ID antigo continua valendo. Solução: confira se existe a linha editandoId = null; dentro da limparFormulario.

!

Os botões Editar e Excluir não respondem

Sintoma: os cards aparecem certos, os botões coloridos estão lá, mas clicar não faz nada. Causa: os addEventListener foram anexados fora da criarCardPet, ou via document.querySelector('.botao-editar') (que pega só o primeiro da página). Solução: ligue os listeners dentro da criarCardPet, usando card.querySelector (com o card como base), depois de definir o innerHTML.

const listaPets = document.querySelector('#lista-pets');
const mensagem = document.querySelector('#mensagem');
const contadorPets = document.querySelector('#contador-pets');

const filtroEspecie = document.querySelector('#filtro-especie');
const botaoSalvar = document.querySelector('#botao-salvar');
const botaoCancelar = document.querySelector('#botao-cancelar');
const campoId = document.querySelector('#pet-id');

let pets = [];
let editandoId = null;

campoBusca.addEventListener('input', mostrarPetsNaTela);
filtroEspecie.addEventListener('change', mostrarPetsNaTela);
botaoCancelar.addEventListener('click', limparFormulario);

formulario.addEventListener('submit', function (evento) {
  evento.preventDefault();
  limparErrosVisuais();

  const pet = montarPetDoFormulario();

  if (erro) { mostrarMensagem(erro, 'erro'); return; }

function cadastrarPet(pet) {
  pets.push({ id: Date.now(), ...pet });
}

function atualizarPet(id, dados) {
  pets = pets.map(function (p) {
    return p.id === id ? { id, ...dados } : p;
  });
}

function montarPetDoFormulario() {
  return {
    nome: document.querySelector('#nome').value.trim(),
    especie: document.querySelector('#especie').value,
    idade: document.querySelector('#idade').value,
    cidade: document.querySelector('#cidade').value.trim(),
    telefone: document.querySelector('#telefone').value.trim(),
    foto: document.querySelector('#foto').value.trim(),
    descricao: document.querySelector('#descricao').value.trim(),
    status: document.querySelector('#status').value
  };
}

function validarPet(pet) {
  if (pet.nome.length < 2) {
    document.querySelector('#nome').classList.add('campo-invalido');
    return 'O nome precisa ter pelo menos 2 letras.';
  }
  if (!pet.especie) {
    document.querySelector('#especie').classList.add('campo-invalido');
    return 'Escolha a espécie do pet.';
  }
  if (pet.idade && Number(pet.idade) < 0) {
    document.querySelector('#idade').classList.add('campo-invalido');
    return 'A idade não pode ser negativa.';
  }
  if (pet.telefone && pet.telefone.length < 8) {
    document.querySelector('#telefone').classList.add('campo-invalido');
    return 'Digite um telefone com pelo menos 8 números.';
  }
  return null;
}

function mostrarMensagem(texto, tipo) {
  mensagem.textContent = texto;
  mensagem.className = 'mensagem ' + tipo;
}

function limparErrosVisuais() {
  const campos = document.querySelectorAll('input, select, textarea');
  for (const campo of campos) {
    campo.classList.remove('campo-invalido');
  }
}

Fechamento

O que levamos desta aula

  • Você implementou as quatro operações do CRUD no front-end: criar, ler, atualizar e excluir.
  • Usou map para atualizar, filter para excluir e buscar, e find para localizar um pet pelo id.
  • Gerou um id único com Date.now() e controlou o modo de edição com a variável editandoId.
  • Ligou eventos a botões criados dinamicamente, com card.querySelector dentro da função que monta o card.
