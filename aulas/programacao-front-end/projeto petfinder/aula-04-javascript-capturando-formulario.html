Seção 01

Introdução

Até agora a página do PetFinder sabe ficar bonita, mas não sabe pensar. Você preenche o formulário, clica no botão e... nada acontece de útil. Hoje isso muda. Vamos criar um app.js e ensinar o navegador a fazer três coisas novas: capturar o clique no botão sem deixar a página recarregar, pegar os dados que o usuário digitou e mostrar cada pet cadastrado como um card abaixo do formulário.

Um aviso honesto desde já: os dados ainda vão ficar só na memória do navegador. Atualizou com F5, sumiu tudo. Isso é esperado — persistência de verdade fica para mais adiante na trilha. Por enquanto, o objetivo é ver o navegador respondendo ao seu código.

O que vamos construir

✓

Ao fim da aula, preencher o formulário e clicar em Cadastrar pet faz nascer um card na tela; o formulário se limpa sozinho, pronto para o próximo. Em concreto: um app.js conectado ao HTML que captura os dados, monta um objeto pet, guarda num array e desenha o pet como card.

O projeto da Aula 03 precisa estar funcionando (HTML semântico + CSS + formulário bonito). E deixe o DevTools aberto na aba Console (F12) o tempo todo: é onde os erros de JavaScript aparecem — e onde você aprende a depurar.

Seção 03

Como o JavaScript conversa com o HTML

i

Conceito-chave

HTML é a estrutura, CSS é a aparência, JavaScript é o comportamento. Pense numa marionete: o HTML é o boneco, o CSS é a roupa, o JS é a mão que faz o boneco se mexer.

Para o JS controlar a página, ele precisa primeiro encontrar os elementos, com document.querySelector() — passando um seletor CSS, o mesmo #pet-form que você usaria no CSS. Depois ele escuta eventos (clique, envio, digitação) com addEventListener(). Quando o evento acontece, uma função é executada — e é dentro dela que você descreve o que muda na página. Encontrar, escutar, reagir: é esse o ciclo que vamos montar.

Seção 04

Passo a passo

1. Atualizar o HTML

No index.html, três ajustes. Primeiro: adicione id="pet-form" na tag <form>. Segundo: depois da section do formulário, dentro de <main>, adicione a área que vai listar os pets:

html 
<section class="area-lista">
  <h2>Pets cadastrados</h2>
  <div id="lista-pets" class="lista-pets"></div>
</section>

Terceiro: logo antes de fechar </body>, adicione a tag que carrega o JavaScript:

html 
<script src="app.js"></script>

2. Estilizar os cards no CSS

No final do style.css, acrescente o estilo da lista, do card, da imagem e do selo de status:

.area-lista {
  margin-top: 30px;
}

.lista-pets {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 20px;
}

.card-pet {
  background: #fff;
  border-radius: 16px;
  overflow: hidden;
  box-shadow: 0 8px 20px rgba(0, 0, 0, 0.08);

  background: #e2e8f0;

  padding: 16px;

  font-size: 14px;

3. Criar o app.js e capturar o submit

Dentro da pasta frontend, crie o arquivo app.js. Este trecho encontra o formulário e a lista, escuta o evento submit e impede o navegador de recarregar a página.

javascript 
const formulario = document.querySelector('#pet-form');
const listaPets = document.querySelector('#lista-pets');
const pets = [];

formulario.addEventListener('submit', function (evento) {
  evento.preventDefault();

  const pet = {
    nome: document.querySelector('#nome').value,
    especie: document.querySelector('#especie').value,
    idade: document.querySelector('#idade').value,
    cidade: document.querySelector('#cidade').value,
    foto: document.querySelector('#foto').value,
    descricao: document.querySelector('#descricao').value,
    status: document.querySelector('#status').value
  };

  pets.push(pet);
  mostrarPetsNaTela();
  formulario.reset();
});

4. Renderizar os pets como cards

Ainda no app.js, logo abaixo, adicione a função que percorre o array e cria um card para cada pet:

javascript 
function mostrarPetsNaTela() {
  listaPets.innerHTML = '';

  for (const pet of pets) {
    const card = document.createElement('article');
    card.classList.add('card-pet');
    const imagem = pet.foto || 'https://placehold.co/600x400?text=Sem+foto';

    card.innerHTML = `
      <img src="${imagem}" alt="${pet.nome}">
      <div class="card-pet-conteudo">
        <h3>${pet.nome}</h3>
        <p>Espécie: ${pet.especie}</p>
        <p>Cidade: ${pet.cidade || 'Sem cidade'}</p>
        <p>${pet.descricao || 'Sem descrição.'}</p>
        <span class="status">${pet.status}</span>
      </div>
    `;
    listaPets.appendChild(card);
  }
}
Seção 05

Checkpoint e desafio

★

Sua página passa se…

Cadastrar um pet faz aparecer um card abaixo do formulário (nome, espécie, cidade, descrição e selo de status); o formulário se limpa sozinho; e três cadastros seguidos geram três cards em grade. Atualizar com F5 apaga tudo — e isso é esperado nesta aula.

★

Desafio (até 8 min)

Personalize a imagem padrão: em vez de ?text=Sem+foto, use ?text=PetFinder na URL do placeholder (troque espaços por +). Depois cadastre um pet sem preencher o campo de foto e confira o resultado.

Seção 06

Erros comuns

!

A página pisca e os dados somem ao clicar

Sintoma: ao clicar em Cadastrar pet, a tela dá um flash, os campos esvaziam, nenhum card aparece e a URL ganha um ? com os dados. Causa: faltou evento.preventDefault() dentro da função do submit. Solução: garanta que essa seja a primeira linha dentro do listener do submit.

!

"Cannot read properties of null" no console

Causa: o navegador executou o app.js antes de o <form> existir, então o querySelector devolveu null. Solução: a tag <script> precisa estar no final do <body>, nunca no <head>. Confira também se o id no HTML é idêntico ao do JS (pet-form com hífen, não petForm).

O que levamos desta aula
