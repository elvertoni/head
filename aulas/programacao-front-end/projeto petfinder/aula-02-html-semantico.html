Seção 01

Introdução

Na Aula 01 o formulário do PetFinder funcionava, mas o HTML era genérico: tudo jogado dentro do <body>, com <br> servindo de espaçador. Hoje reescrevemos a mesma página usando HTML semântico — tags que dizem o papel de cada parte. É o passo entre escrever HTML "que funciona" e escrever HTML que outro programador, um leitor de tela ou o próprio Google conseguem entender.

Seção 02

O que vamos construir

✓

Entrega da aula

A mesma página da Aula 01, reescrita com tags semânticas — <header> no topo, <main> no centro, <section> envolvendo o formulário. O formulário ganha campos profissionais: lista suspensa de espécies (<select>), descrição em caixa de texto (<textarea>), campo de URL para a foto e campo específico de telefone. Pelo menos dois campos passam a ter required — o navegador bloqueia o envio se estiverem vazios.

i

Antes de começar

Você precisa do index.html da Aula 01 já criado dentro da pasta frontend. Se faltou na aula passada, peça o arquivo pronto antes de continuar. Mantenha o VS Code aberto e o navegador pronto para recarregar com F5.

Seção 03

Por que HTML semântico

i

O que significa "semântico"

Semântico vem de "sentido". Uma tag semântica diz o papel daquele pedaço de página, não só a aparência. Uma <div> serve para tudo; um <header> diz "este é o cabeçalho"; um <main> diz "este é o conteúdo principal"; um <section> diz "esta é uma seção temática".

Isso ajuda o navegador, o leitor de tela do colega cego, o buscador do Google e o próximo programador que abrir o código — inclusive você mesmo daqui a uma semana. Mesma página na tela, significado completamente diferente por baixo.

Seção 04

Passo a passo

Vamos substituir todo o conteúdo entre o <body> de abertura e de fechamento. Antes de apagar, leia os três passos até o fim.

1. Trocar o esqueleto pelo HTML semântico

Apague tudo entre <body> e </body>. No lugar, digite a estrutura abaixo — repare nas três tags novas: header, main e section.

html 
<body>
  <header>
    <h1>PetFinder Escolar</h1>
    <p>Cadastre pets disponíveis para adoção na sua cidade.</p>
  </header>

  <main>
    <section>
      <h2>Cadastro de Pet</h2>

      <!-- formulário entra aqui no próximo passo -->

    </section>
  </main>
</body>

2. Primeira metade do formulário

No lugar do comentário, digite o bloco abaixo. O <select> gera a lista suspensa, e o required obriga o preenchimento.

html 
<form>
  <div>
    <label for="nome">Nome do pet:</label>
    <input type="text" id="nome" name="nome" placeholder="Ex: Luna" required>
  </div>

  <div>
    <label for="especie">Espécie:</label>
    <select id="especie" name="especie" required>
      <option value="">Selecione</option>
      <option value="Cachorro">Cachorro</option>
      <option value="Gato">Gato</option>
      <option value="Coelho">Coelho</option>
      <option value="Outro">Outro</option>
    </select>
  </div>

  <div>
    <label for="idade">Idade aproximada:</label>
    <input type="number" id="idade" name="idade" min="0" placeholder="Ex: 2">
  </div>

3. Segunda metade do formulário

Entre o </div> do campo idade e o </form>, adicione os campos abaixo. Aqui aparecem dois tipos novos de input (url e tel) e a <textarea>, que cria uma caixa de texto grande.

html 
  <div>
    <label for="porte">Porte:</label>
    <select id="porte" name="porte">
      <option value="">Selecione</option>
      <option value="Pequeno">Pequeno</option>
      <option value="Médio">Médio</option>
      <option value="Grande">Grande</option>
    </select>
  </div>

  <div>
    <label for="cidade">Cidade:</label>
    <input type="text" id="cidade" name="cidade" placeholder="Ex: Curitiba">
  </div>

  <div>
    <label for="responsavel">Nome do responsável:</label>
    <input type="text" id="responsavel" name="responsavel">
  </div>

  <div>
    <label for="telefone">Telefone:</label>
    <input type="tel" id="telefone" name="telefone" placeholder="Ex: 41999998888">
  </div>

  <div>
    <label for="foto">URL da foto:</label>
    <input type="url" id="foto" name="foto" placeholder="https://exemplo.com/foto.jpg">
  </div>

  <div>
    <label for="descricao">Descrição:</label>
    <textarea id="descricao" name="descricao" rows="4" placeholder="Conte um pouco sobre o pet"></textarea>
  </div>

  <div>
    <label for="status">Status:</label>
    <select id="status" name="status">
      <option value="Disponível">Disponível</option>
      <option value="Adotado">Adotado</option>
    </select>
  </div>

  <button type="submit">Cadastrar pet</button>
</form>
Seção 05

Checkpoint e desafio

★

Sua página passa se…

O cabeçalho aparece com título e texto explicativo; o formulário tem dez campos na ordem nome, espécie, idade, porte, cidade, responsável, telefone, URL da foto, descrição e status; tentar enviar com o campo Nome do pet vazio dispara a mensagem de erro do próprio navegador; e a lista de espécie traz Selecione como primeira opção.

★

Desafio (até 12 min)

Adicione duas espécies ao <select>: Hamster e Pássaro. Depois crie um campo Sexo, do tipo select, com as opções Macho e Fêmea, posicionado logo após o campo Porte. Recarregue e confira se as opções aparecem corretamente.

Seção 06

Erros comuns

!

O for do label não bate com o id do input

Sintoma: clicar no texto do rótulo não coloca o cursor dentro do campo. Causa: você escreveu <label for="nome"> mas o input tem id="nomePet" — a ligação quebra. Solução: o valor do for precisa ser idêntico ao id do input, letra por letra, maiúsculas incluídas.

!

Esquecer o option de valor vazio no select

Sintoma: mesmo sem o aluno escolher a espécie, o formulário envia "Cachorro", porque era a primeira opção. Causa: o primeiro option tinha valor preenchido e fica selecionado por padrão. Solução: deixe a primeira opção como <option value="">Selecione</option>. Combinada com required, ela força o usuário a escolher.

Fechamento

O que levamos desta aula

  • Você diferenciou HTML genérico de HTML semântico, usando header, main e section.
  • Conheceu novos tipos de input (number, tel, url) e duas tags novas: select e textarea.
  • Tornou campos obrigatórios com o atributo required e entendeu o papel do option vazio.
  • Na próxima aula vamos ligar um style.css e transformar essa página branca num formulário com cara de aplicação de verdade.
