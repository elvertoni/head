# Brief de imagem — Aula 23: Introdução à Arquitetura de Computadores - Parte 1

> Neutro de plataforma. Imagem só entra com justificativa pedagógica (teste anti-decoração). Final web ≤500 KB em `img/`; original pesado fica no `lake/` (fora do git).

## 1. Computador como cidade da informação

- **secao:** Abertura / Desenvolvimento › O computador como sistema
- **objetivo:** Mostrar que o computador não é uma peça isolada, mas uma organização de partes com fluxo: entrada, processamento, memória, armazenamento e saída. Serve como âncora visual para a aula inteira.
- **alt:** Ilustração de um computador visto como uma pequena cidade, com setores identificados como entrada, CPU, RAM, armazenamento, barramentos e saída, conectados por ruas por onde circulam dados.
- **prompt:** Educational flat vector illustration, 16:9. A desktop computer represented as a small city map: an input gate with keyboard/mouse icons, a CPU building processing tasks, a RAM workspace, a storage warehouse, output screen/speaker area, and roads/buses labeled as data paths connecting everything. Clear labels in Portuguese: Entrada, CPU, RAM, Armazenamento, Barramentos, Saída. Bright but professional colors, no tiny text, clean classroom style.

## 2. RAM x armazenamento

- **secao:** Desenvolvimento › As peças principais
- **objetivo:** Fixar a diferença entre memória temporária e armazenamento persistente, um erro recorrente em turmas iniciantes.
- **alt:** Mesa de trabalho com papéis abertos representando RAM ao lado de um armário/arquivo representando armazenamento permanente.
- **prompt:** Simple educational illustration, 16:9. Split scene: left side a work desk labeled "RAM" with open papers, sticky notes and active tasks; right side a filing cabinet labeled "Armazenamento" with folders saved for later. Add a small power-off icon showing the desk clears but cabinet remains. Flat vector, clear Portuguese labels, no long paragraphs.

## 3. (Opcional) Fluxo entrada-processamento-saída

- **secao:** Fallback visual do `diagrama-progressivo`
- **objetivo:** Ter uma versão estática do fluxo caso o renderer usado não suporte diagrama progressivo.
- **alt:** Fluxo em cinco blocos: Entrada → CPU → RAM ↔ Armazenamento → Saída.
- **prompt:** Minimal flow diagram, 16:9. Five connected blocks with icons and Portuguese labels: Entrada, CPU, RAM, Armazenamento, Saída. Show RAM and Armazenamento connected both ways. Use arrows, clean white background, high contrast, classroom-friendly design.
